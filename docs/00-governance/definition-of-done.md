# Definition of Done

A task is done only when:

- code compiles;
- relevant tests/checks pass;
- acceptance criteria are satisfied;
- traceability is updated when behavior changed;
- architecture boundaries are preserved;
- no undocumented API/DB/event/permission contract change is introduced;
- no unrelated refactor is included;
- no unapproved dependency is added;
- error handling follows the documented convention;
- tenant, workspace, store, auth, and permission behavior are correct;
- frontend screens handle required loading/empty/error/success states;
- documentation is updated when behavior, architecture, or contracts changed.

The final handoff must include:

- summary;
- changed files;
- validation commands and result;
- skipped validation with reason;
- risks or follow-up.
