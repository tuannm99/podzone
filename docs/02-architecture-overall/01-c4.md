# C4 Architecture Index

This file is the C4 entrypoint for Podzone.

## C1: System Context

Read:

- `system-context.md`

Purpose:

- identify users, external systems, and Podzone as a system.

Use before:

- changing actors;
- adding external integrations;
- changing product boundary.

## C2: Container View

Read:

- `containers.md`

Purpose:

- identify deployable services, databases, queues, gateways, and frontend
  containers.

Use before:

- adding a service;
- changing service communication;
- changing deployment topology.

## C3: Component/Module View

Read:

- `modules.md`
- `bounded-contexts.md`
- `data-ownership.md`

Purpose:

- identify modules inside services and data ownership boundaries.

Use before:

- refactoring service internals;
- applying DDD;
- changing repository ownership;
- adding cross-service calls.

## C4: Code/Sequence Details

Read:

- `code-map.md`
- `sequences.md`
- `transport-contracts.md`
- `async-messaging.md`

Purpose:

- explain code packages, important flows, transport contracts, and async event
  flow.

Use before:

- implementing a vertical slice;
- changing generated contracts;
- changing worker/outbox/event behavior.

## Required Traceability

Every architecture-significant sprint task should link:

```text
SRS requirement
  -> C4 doc
  -> contract or DB note
  -> sprint task
  -> verification result
```
