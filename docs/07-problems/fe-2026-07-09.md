## New Findings

### NEW1 — `createTenantOrdersViewModel.ts:163` — `recommendationResource` unstable object source (MEDIUM)

Same class as the `queuePageResource` bug fixed in `FIX.md Part 6`. The source
returns a new object literal on every reactive evaluation:

```ts
const [recommendationResource] = createResource(
    () => {
        if (!workspaceReady()) return undefined
        const candidateID = orderForm.values.selectedCandidateId.trim()
        if (!candidateID) return undefined
        return {
            candidateId: candidateID,
            productType: orderForm.values.selectedProductType,
            shipRegion: orderForm.values.selectedShipRegion,
            preferredPartner: orderForm.values.preferredPartner.trim() || undefined,
        }
    },
    async (source) => getRoutedOrderRecommendation(source)
)
```

`createResource` source comparison uses `===`. Every reactive read that touches
any of the tracked signals (workspace signals, `orderForm.values.*`) produces a
new object → comparison always fails → spurious refetch even when field values
are unchanged.

**Fix:** Serialize to a stable string key; fetcher reads form values directly
(untracked in fetcher context):

```ts
const [recommendationResource] = createResource(
    () => {
        if (!workspaceReady()) return undefined
        const candidateID = orderForm.values.selectedCandidateId.trim()
        if (!candidateID) return undefined
        return [
            candidateID,
            orderForm.values.selectedProductType,
            orderForm.values.selectedShipRegion,
            orderForm.values.preferredPartner.trim(),
        ].join('|')
    },
    async () =>
        getRoutedOrderRecommendation({
            candidateId: orderForm.values.selectedCandidateId.trim(),
            productType: orderForm.values.selectedProductType,
            shipRegion: orderForm.values.selectedShipRegion,
            preferredPartner: orderForm.values.preferredPartner.trim() || undefined,
        })
)
```

**File:** `internal/ui-podzone/src/modules/backoffice/pages/orders/createTenantOrdersViewModel.ts:163`

### NEW2 — `createTenantOrdersViewModel.ts:314` + `app-router.tsx:121` — URL sync is one-way, orders route missing `validateSearch` (MEDIUM, same root as Part 8)

There is a `createEffect` syncing URL search params (`search().queueView`,
`search().queueSort`, `search().operatorLens`) into local signals, but the
reverse direction is not implemented: `setActiveQueueView(v)` etc. update only
the local signal, not the URL. Refreshing the page loses all queue
view/sort/lens/page/search state.

Additionally, `tenantOrdersRoute` in `app-router.tsx:121` has **no
`validateSearch`** schema. TanStack Router treats `queueView` etc. as `unknown`,
defeating type safety and causing the one-way sync to silently no-op on initial
load (the router strips unvalidated params).

**Rule violated:** `agent/SOLID_STYLE_GUIDE.md` lines 283–284 — collection page/filter/sort
must be URL state, not `createSignal`.

**Signals to migrate** (all 6 are currently `createSignal` in `createTenantOrdersViewModel.ts`):
`queuePage`, `queueSearch`, `appliedQueueSearch`, `activeQueueView`, `activeQueueSort`, `operatorLens`.

**Implementation steps:**

1. Add `validateSearch` to `tenantOrdersRoute` in `app-router.tsx:121`:
   ```ts
   validateSearch: (search) => ({
       queueView: isQueueView(search.queueView) ? search.queueView : 'all',
       queueSort: isQueueSort(search.queueSort) ? search.queueSort : 'createdAt',
       operatorLens: typeof search.operatorLens === 'string' ? search.operatorLens : '',
       queuePage: typeof search.queuePage === 'number' ? search.queuePage : 1,
       queueSearch: typeof search.queueSearch === 'string' ? search.queueSearch : '',
   })
   ```

2. In `createTenantOrdersViewModel.ts`, replace local signals with URL reads/writes:
   ```ts
   const search = useSearch({ from: '/t/$tenantId/orders' })
   const navigate = useNavigate()

   const activeQueueView = () => search().queueView
   const setActiveQueueView = (v: QueueView) =>
       void navigate({ search: (prev) => ({ ...prev, queueView: v }) })
   // same for activeQueueSort, operatorLens, queuePage, queueSearch
   // appliedQueueSearch becomes: () => search().queueSearch (applied on navigate, no separate signal)
   ```

3. Remove the one-way sync `createEffect` at line 314.

4. Remove the 6 now-unused `createSignal` declarations.

Note: this is a larger refactor spanning route schema + ViewModel + View. Handle in a
dedicated sprint slice, not a hotfix.

**Files:**
- `internal/ui-podzone/src/modules/backoffice/pages/orders/createTenantOrdersViewModel.ts`
- `app-router.tsx` (add `validateSearch` to `tenantOrdersRoute`)
